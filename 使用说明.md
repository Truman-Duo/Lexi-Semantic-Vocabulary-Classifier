# Lexi 词汇分类工具 — 使用说明

## 简介

Lexi 是一款离线多标签英文词汇意群分类工具。输入英文文本，自动将单词归入三级分类体系（主观类 / 客观类 / 抽象类）及其 10 个子类，输出词频排序的词汇表。

```
总输出
├─ 主观类（和"人"相关的）
│  ├─ 情绪感受     ── emotion, joy, fear, love ...
│  ├─ 观点判断     ── judgment, belief, opinion ...
│  ├─ 心理活动     ── cognition, memory, decision ...
│  └─ 主观动作     ── action, walk, communicate ...
│
├─ 客观类（和"世界"相关的）
│  ├─ 具体事物     ── artifact, animal, building ...
│  ├─ 客观动作     ── happen, occur, change ...
│  └─ 属性特征     ── attribute, color, quality ...
│
├─ 抽象类（概念性的）
│  ├─ 基础概念     ── concept, quantity, time ...
│  ├─ 社会概念     ── society, government, economy ...
│  └─ 关系连接     ── relation, cause, condition ...
│
└─ 过滤词（纯功能词）
   └─ 停用词       ── the, a, in, of ...
```

## 安装

### 1. 安装 Python

需要 Python 3.8 或更高版本。[python.org](https://www.python.org/downloads/) 下载安装。

### 2. 安装依赖包

在项目目录下打开终端（命令提示符），运行：

```bash
pip install lemminflect wordfreq nltk
```

可选依赖（用于 Anki 牌组导出）：

```bash
pip install genanki
```

### 3. 下载 NLTK 数据

```bash
python -c "import nltk; nltk.download('wordnet'); nltk.download('omw-1.4'); nltk.download('averaged_perceptron_tagger_eng')"
```

### 4. 构建分类词库

首次使用前需要构建分类词库（基于 WordNet 上义词路径，耗时 5-10 分钟）：

```bash
python build_full_categories.py
```

生成 `data/categories_full.json`（约 15.5 万词，4.2MB）。

> 如果使用 `run_lexi.bat`，词库不存在时会自动构建，无需手动执行此步骤。

## 快速开始

### GUI 模式（推荐新手）

**方法一：** 双击 `run_lexi.bat`（无参数启动）

**方法二：** 在终端运行

```bash
python gui.py
```

GUI 操作步骤：

1. 点击"输入文件"旁的「浏览」选择英文文本（如 `test.txt`）
2. （可选）调整分类词库、停用词表路径
3. （可选）勾选需要的输出格式（Markdown / JSON / CSV / HTML / Anki）
4. 点击「开始分类」
5. 等待处理完成，查看结果或点击「打开输出文件夹」

### CLI 模式

```bash
python cli.py 你的文件.txt
```

此命令会在当前目录生成 `output.md` 和 `output.json`。

常用选项：

```bash
# 同时导出 CSV 和 HTML 交互浏览器
python cli.py 你的文件.txt --output-csv output.csv --output-html output.html

# 导出 Anki 牌组（需安装 genanki）
python cli.py 你的文件.txt --output-anki deck.apkg

# 使用自定义分类覆盖
python cli.py 你的文件.txt --overrides my_overrides.json

# 指定输出路径（默认在当前目录）
python cli.py 你的文件.txt --output-md ./结果/report.md --output-json ./结果/data.json
```

### Windows 拖放模式

将文本文件拖拽到 `run_lexi.bat` 上释放，自动生成 CSV 和 HTML 输出。

## 输出格式说明

### Markdown (`.md`)

按分类层级组织的编号列表，末尾附统计摘要：

```
# 主观类

## 情绪感受
1. love [A1]
2. fear [A2]
3. anxiety [B1]
...
---
## 统计摘要
- **单词总数**: 4521
- **分类方法分布**:
  - dictionary: 3718 (80.0%)
  - pos_fallback: 679 (14.6%)
  - suffix: 185 (4.0%)
  - fuzzy: 49 (1.1%)
  - stopword: 15 (0.3%)
- **客观类**: 4785 词 (52.6%)
- **抽象类**: 2703 词 (29.7%)
...
```

单词后的 `[A1]` 表示 CEFR 难度等级（A1 最简单 → C2 最难）。

### JSON (`.json`)

结构化数据，每个词附带元信息：

```json
{
  "total_words": 4521,
  "categories": {
    "主观类": {
      "情绪感受": [
        {
          "word": "love",
          "zipf": 4.89,
          "cefr": "A1",
          "confidence": 1.0,
          "source": "dictionary"
        }
      ]
    }
  }
}
```

- `zipf`: Zipf 词频分数（越高越常见）
- `cefr`: 欧洲共同语言参考标准等级
- `confidence`: 分类置信度（1.0=精确匹配, 0.8=派生匹配, 0.6=后缀规则, 0.4=词性回退）
- `source`: 分类来源（dictionary / fuzzy / suffix / pos_fallback）

### CSV (`.csv`)

可用 Excel 或任意表格软件打开，列包括：word, main_category, sub_category, zipf_frequency, cefr_level, confidence, source

### HTML 交互浏览器 (`.html`)

自包含的网页（无需网络），功能：
- 左侧分类树，点击展开子类
- 搜索框按单词过滤
- CEFR 等级筛选下拉框
- 词频 / 字母排序切换
- 单词右侧显示 CEFR 标签和分类来源

直接用浏览器打开即可。

### Anki 牌组 (`.apkg`)

可导入 Anki 软件进行间隔重复记忆：
- 正面：单词
- 背面：分类（主观类/客观类/抽象类 > 子类）、CEFR 等级、词频

需要安装 `genanki`：`pip install genanki`

## 高级功能

### 自定义分类覆盖

创建 JSON 文件（如 `my_overrides.json`），格式与 `categories_full.json` 相同：

```json
{
  "主观类": {
    "情绪感受": ["serendipity", "wanderlust"],
    "心理活动": ["flow", "mindfulness"]
  },
  "客观类": {
    "具体事物": ["smartphone", "bitcoin"]
  }
}
```

使用：

```bash
python cli.py test.txt --overrides my_overrides.json
```

覆盖规则优先级最高，会覆盖词典内置分类。

### 多标签分类

一个单词可能同时属于多个类别。例如 "light" 既可能是**客观类/属性特征**（重量轻的），也可能是**主观类/主观动作**（点燃）。输出中会在不同类别下同时出现。

### 派生词智能匹配

当词典中没有某个词时，系统会自动剥离常见派生后缀并重新查找。例如 "happiness" → "happy"、"readable" → "read"、"activation" → "activate"。匹配到的词标注置信度 0.8，来源为 `fuzzy`。

## 文件结构

```
lexi/
├── cli.py                      # 命令行入口
├── gui.py                      # 图形界面

├── build_full_categories.py    # 分类词库构建脚本
├── lexi/                       # 核心代码包
│   ├── pipeline.py             # 主流程 + 所有输出格式
│   ├── cleaner.py              # 文本清洗
│   ├── lemmatizer.py           # 词形还原
│   ├── classifier.py           # 分类引擎
│   ├── sorter.py               # 排序 + CEFR
│   └── models.py               # 数据模型
└── data/
    ├── categories_full.json    # 分类词库
    └── stopwords.txt           # 停用词表
```

## 常见问题

**Q: 提示 "Missing dependency 'lemminflect'"**

A: 运行 `pip install lemminflect`

**Q: 提示 "文件不存在: data/categories_full.json"**

A: 运行 `python build_full_categories.py` 生成分类词库

**Q: 终端输出中文乱码**

A: Windows 终端对 UTF-8 中文支持有限，不影响实际输出文件（文件内容编码正确）

**Q: Anki 导出失败**

A: 需要额外安装 `pip install genanki`

**Q: 如何处理超长文本？**

A: Lexi 按行流式读取，内存占用可控。4521 个唯一词的处理时间约 10-30 秒。
